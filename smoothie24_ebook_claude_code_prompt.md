## 📋 프로젝트 개요

Smoothie24(스무디24) 무인 생과일 스무디 머신 프랜차이즈의 **예비 점주 모집 전자책**을 HTML 단일 파일로 제작한다. 예비 창업자가 처음 링크를 열었을 때 "이 사업은 전문적이고 신뢰할 수 있다"는 인상을 즉시 받을 수 있어야 한다.

GitHub Pages로 배포되는 정적 사이트 구조(`index.html` + `image/`)이며, **총 11페이지**가 구현·연결되어 있다.

```css
--color-primary:     #E8316A   /* Smoothie Pink */

--color-text-sub:    #555577

```

**배경 장식 숫자(각 페이지 우상단 대형 반투명 숫자)** — 전 페이지 `clamp(160px, 22vw, 260px)` / `top:-0.12em; right:-0.08em`로 통일되어 있다 (과거 일부 페이지만 `clamp(180px,32vw,360px)`를 쓰던 불일치를 통일 완료).

**헤더 라이트/다크 테마** — CI 로고는 기본적으로 흰색(`filter:brightness(0) invert(1)`)이며, 배경이 밝은 페이지에서만 원색으로 표시된다:
```js
const LIGHT_PAGES = new Set([1, 2, 3, 5, 6, 7]); // data-page 기준: 시장성·제품·기술·이용방법·운영·입지
```

### PAGE 01 — 표지

### PAGE 02 — 시장성

카드 3개(인건비 절감/24시간 매출/건강 트렌드), 마우스 호버 시 테두리 색 변화 **제거됨**(클릭 가능 오해 방지).

### PAGE 03 — 원재료 공급

과일컵 2종에 **"과일 함유 40%"/"70%"** 배지 추가. 하단 배지 3종: "리얼 과일 100% 사용" / "얼음 Zero · IQF 급속냉동 과일 사용" / "본사 일괄 생산 · 콜드체인 직배송으로 품질 균일화".

### PAGE 04 — 스마트 머신 사양

- 제목 "BLENIQ" → **"과일 스마트 스무디 머신"**으로 변경, 머신 이미지도 `machine-pink.png`(핑크 컬러)로 교체
- 치수선(260mm/670mm) 그래픽 **삭제**, 대신 하단에 "* 기본 사양: 핑크 / 협의 시 컬러 커스터마이징 가능" 캡션
- 스펙 카드 4개(10.1인치/25,000RPM/원격관리/500ml) + **하단 "상세 스펙" 아이콘 배지 5종**(크기·중량 / 전기사양 / 제조용량 / 세척 / 통신인터페이스) — 색상은 단일 톤(중립 회색), 아이콘으로만 구분
- 제조 시간: 블렌딩 120초 + 세척 60초 = **총 180초/컵**, 제조 용량 **20컵/1시간**로 확정 (PAGE 07의 "50~60초" 표기와 정합성 맞춤)

### PAGE 05 — 위생 관리

3-Phase 순환 다이어그램 + 컵 물 채워지는 애니메이션은 기존과 동일. **60초 자동 세척 진행 링**(연한 로즈핑크 `#F28FAE`, 4.5초간 카운트업 + 0.5초 유지 후 반복) + 하단 큰 숫자 카운터 신규 추가.

### PAGE 06 — 이용방법 (신규 페이지)

- 헤더: eyebrow "How to Use" / 제목 "스무디머신 이용방법" / 부제 "구매부터 제조 완료까지, 누구나 쉽게 5단계!"
- 3단 본문: 좌측 `freezer-photo.png` 카드("냉동 컵 선택") — 중앙 5단계 리스트(핑크 원형 숫자 + "⏱ 약 120초 완성" 배지 + 5번 항목 아래 "자동세척을 위해서 문을 꼭 닫아주세요!" 경고) — 우측 `machine-dispense.png` 카드("완성품 수령")
- 모바일에서는 사진→리스트→사진 순서로 자연 스택 (DOM 순서 그대로 유지)
- 하단 안내 바(`.loc-info-bar` 재사용) + 태그 4종(`.loc-card-tag` 재사용)

### PAGE 07 — 운영 프로세스

STEP 1(아침 점검, **약 20분**) / STEP 2(완전 자동 운영, 24시간) / STEP 3(저녁 마감, **약 10분**) — 총 30분은 동일. STEP1 체크리스트 3번째 항목이 "앱으로 전날 매출 원격 확인" → **"위생을 위한 판매 전 강력 자동 세척 실시"**로 변경. STEP2 블렌딩 표기가 **"120초 자동 블렌딩 완성"**(자동세척 별도) + **"사용 후 60초간 온수 자동 세척"**로 분리 표기.

호버 시 테두리 색 변화 **제거됨**. 배경 장식 숫자("08") 신규 추가(과거 누락 상태였음).

### PAGE 09 — 초기 창업 비용

- 좌: 헤더 + 비용표(리스 헤더가 오른쪽 리스 박스와 **같은 높이에서 시작**하도록 레이아웃 재구성 — `.invest-left` wrapper로 헤더+표를 묶어 그리드 좌측 컬럼화)
- 표 총액 **12,650,000원 (VAT포함)** — 스무디 머신 1대 6,050,000원 + 냉동고(키오스크 포함) 1대 6,600,000원, "판매 부스(하우징)" 명칭 통일
- 우: **"36개월 리스 프로그램" 2단 비교 카드** — 상단 VAT 안내 문구, 플랜A(보증금·잔존가치 20%, 연 7.60%, 월 331,280원) / 플랜B(30%, 연 6.80%, 월 294,110원, 기본 선택), 공통 안내 2줄, "두 플랜 비교" 요약(보증금/월리스료/3년 총현금유출 차이)
- "할부"라는 표현은 전부 "리스"로 통일 (단, PAGE 10 시뮬레이터의 "월 할부금" 라벨은 아직 미수정 상태로 남아있음 — 하단 참고)

### PAGE 10 — 수익 시뮬레이터 (대폭 개편)

기존 단일 계산 구조 → **탭 2개로 분리**:

**공통 입력**: 판매단가(3,000~5,000원) / 하루 판매량(**기계 1대당**, 20~80잔로 확장) / **기계 대수**(+/− 스테퍼, 1~10대, 신규)

**직접운영형 탭**: 리스 플랜 토글(20%/30%, 기본 30%) + 비용 내역 박스. 결과: 월매출/월총비용/월순수익(손익분기 색상 코딩 유지)/연간기대수익/하루순이익/**손익분기 하루판매량(신규)**.

**파트너 제휴형 탭**: 수익쉐어율 슬라이더(**30~60%**, 5% 단위, 기본 50%). 결과: 월매출/수익쉐어율/**"제휴처 월 예상 수익"**(월순수익이 아님 — 비용 차감 없는 배분액이라는 점 라벨로 명시)/연간기대수익/하루예상수익/기계구입비 0원·기계임대료 0원(고정 표시).

계산 상수: `LEASE_PER_UNIT_20=331280`, `LEASE_PER_UNIT_30=294110`(PAGE 09 확정 리스료 반영), `OPEX=250000`(1대당, 대수 곱연산).

> ⚠️ **잔여 이슈**: 이 페이지의 비용 내역 박스 라벨이 아직 `"월 할부금 (기계 리스료)"`로 남아있어 "할부"라는 옛 표현이 하나 남아있다. PAGE 09에서 "리스"로 완전히 용어 통일했으므로, 다음 작업 시 `"월 리스료"`로 맞추는 게 좋다.
